ID Document Capture


A Thales tool call IDV

Current version: 4.5.0

License required, to be provided at SDK initialization.

The Seventh Edition of Doc 9303 represents a restructuring of the ICAO specifications for Machine Readable Travel Documents. Without incorporating substantial modifications to the specifications, in this new edition Doc 9303 has been reformatted into a set of specifications for Size 1 Machine Readable Official Travel Documents (TD1), Size 2 Machine Readable Official Travel Documents (TD2), and Size 3 Machine Readable Travel Documents (TD3), as well as visas.

This set of specifications consists of various separate documents in which general (applicable to all MRTDs) as well asMRTD form factor specific specifications are grouped.

  • TD1: 53.98 mm x 85.6 mm (2.13 in x 3.37 in).
  • TD2: 74.0 mm x 105.0 mm (2.91 in x 4.13 in).
  • TD3: 88.0 mm x 125.0 mm (3.46 in x 4.92 in).

This tool will help you to Capture official Document: Passport, ID card (Front and Back) or all ICAO documents like Driving license.

Additionally sdk offer Auto Document Type Detection capture, it will help to detect the document type automatically while scanning and decide to capture front and back side of the ICAO document's.

The SDK is having for that 4 Captors that will be used automatically when your are using the API session.resume(taskID) and we offer some customization.


Captors

val tdiMobileSdk = TdiMobileSdk.getInstance()
tdiMobileSdk.captorProvider["Passport"]       //Passport Capture (1 page) - TD3
tdiMobileSdk.captorProvider["ID Card"]        //ID Document Capture (2 pages) - TD1-TD2
tdiMobileSdk.captorProvider["ICAO"]           //ICAO like Driving license (1 or 2 pages) - TD1-TD2-TD3
tdiMobileSdk.captorProvider["AutoDetectDocIDV"]  //Auto document type detection on all ICAO documents like Passport/ID Card/Driving license (1 or 2 pages) - TD1-TD2-TD3

Guiding frame

  • true default value, to display a Guiding frame to help user to fit the document during cature

  • false to hide the capture frame

Document sides count

This value set the number of document side(s) to capture, e.g. 1 for a passport, 2 for an Identity card

Time out

By default set at 0 meaning no Time out. This timer value in milliseconds can be a security feature, by setting a time frame to execute the document capture before aborting to limit people trying fake capture.

Edge Detection Mode

To locate a document on the camera the SDK offers different approaches that can suit in different situations.

  • MachineLearning is the default method. It internally uses a Machine Learning algorithm trained with thousand of documents. This method is the default value and it is the recommended method.

  • ImageProcessing is using pure image processing algorithms to find the edges of a document. This methods provides more accuracy and performance but requires better environment conditions.

Blur Detection

An image is considered blur if it is not clearly defined, not having distinct in outline or detail.

Factors that might affect the assessment vary, from lighting conditions to document colors and details.

A darker environment or a less detailed document will tend to blur conditions.

TextBlock is the default value for this check.

Strict and Relaxed modes are the legacy mechanisms that try to measue the blurrines of the whole picture whereas TextBlock focus only on the text regions.

  • TextBlock is the default value for this check.

  • Strict and Relaxed modes are the legacy mechanisms that try to measue the blurrines of the whole picture whereas TextBlock focus only on the text regions.

  • Disabled, Disable the Blur detection (can generate image quality issue)

Glare Detection

A glare usually happens when a direct spotlight focuses on a reflective surface.

This might impact the visualization of the document content and must be avoided.

Recommended lighting conditions are having even ambient lights over the document.

  • Color This mode detects colored light glares (including holograms) making it more strict that the white mode.

  • White default value, mode is only able to detect glares on the grayscale, meaning that only detects white light glares.

  • Disabled, Disable the Glare detection (can generate image quality issue)

Darkness Detection

The aim of this check is to detect dark environment conditions which usually cause a poor data extraction.

This might impact the visualization of the document content and must be avoided.

  • Relaxed is the default value.

  • Disabled, Disable the Darkness detection (can generate image quality issue)

Photocopy Detection

This check will check if the detected document is a photocopy.

  • BlackAndWhite is the default value. It analyzes the absence of color.

  • Disabled, Disable the detection

Take into account that some documents lack of coloured details causing false positive warnings.

Skippable

  • true to display the SKIP button to allow user to skip the capture

  • false default value, to hide the SKIP button, user must process the capture

This setting is overrided by the Profile API

Shutter Button Delay

Default value is -1, no shutter button will be shown by default. The purpose of this button is to help end user in difficult environment that won't allow the automatic capture. This button can be display if you need after n second.

If you decided to use this, please provide Int value in seconds

When the Shutter button is displayed, the automatic capture is still ON, if the condition are good, the photo will be taken without any action on the shutter.

If you click on the shutter button, it will force the photo to be taken regarless of the conditions (blur, glare, darkness, edge detection). Our sdk will try to find the document:

​ a. Document is found, the document will be cropped at the edges and you will see the result to be comfirmed.

​ b. Document isn't found, the image will be cropped by default using the guidance frame (if this one is set) and you will see the result which you can manual edit the cropping if the result isn't satifying.

For both case a and b you will be notify if the blur quality seems not right for our backend to read correctly the image.

⚠ WARNING:
The usage of manual capture and/or manual cropping could lead to a lower photo quality for the Backend to do the OCR and check the integrity of the document.

Capture camera orientation

  • 0 default value, Landscape, the capture is done in landscape mode

  • 1 Portrait, the capture is done in portrait mode

  • 10 Automatic, the capture follow the device rotation (if enabled by user on device settings)

Captor Callback

val captorCallback = object : Captor.Callback{
    override fun onDataCaptured(input: Input) {
        input.pages["page1"] // Front Document
        input.pages["page2"] // Back Document (if any)
    }

    override fun onFailure(failure: Failure) {
        // Capture failed, see Error guide chapter to identify what's the reason.
    }
}

Captor.Callback is an optional config to help you to:

  1. Collect the data captured during a call of resume() API. Those data are automatically sent to server by TDI sdk after Capture, what you get here, is only for you if you need. Please be sure to keep it safe.

  2. Listening what's happening on Failure scenario, see Error guide Chapter for reference. It could be a simple Cancel by pressing Back button or tool issue.

Camera Mode

This will set the camera capture mode during camera object initialization. Below are the valid values.

  • 0 - Default (0) is the default value. It returns the images taken directly from the processing stream balancing between quality and speed.

  • 1 - HighQuality (1) returns the image with specific camera settings focusing in quality rather than speed and size.

Ease the capture

Set the captor mode to ease the ID doc capture by changing detection settings depending of light conditions and user action.

  • false - disabled is the default value, the mechanism is disabled

  • true - enabled. Ease the capture mechanism is activated to change the detection settings when capturing a document is difficult.

The mechanism will change default detection settings depending of capture conditions using thresholds and timing windows.

e.g. When no document is automatically captured after a delay the captor will change settings from Strict or Relaxed to disabled.

Affected settings priority: Glare, Blur, Focus Dark and at the end switch of Edge detection mode

This mechanism helps to trig a capture but can result document image issue like hot spots (glare) or bad focus.

First, it is better to improve capture conditions, by changing light environment or document position. If it is not enough, this mechanism can be activated.
