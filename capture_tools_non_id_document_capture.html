Non ID Document Capture


A Thales tool by TDI

Current version: 1.0.0

No License required

This tool will help you to Capture any type of document size like a bill or a ticket.

This captor allows to:

  • Capture a picture using the handset camera
  • Try to detect automatically the document edges
  • Manuel rotation and cropping adjustment if required


val tdiMobileSdk = TdiMobileSdk.getInstance()
tdiMobileSdk.captorProvider["Non-ID"]


Customization

See Non ID document captor presentation for more information about customization

Captors configuration:

â€‹ Captors can get a config Map to set different capture parameters.

TdiMobileSdk.getInstance().tdiMobileSdk.captorProvider["Non-ID"]?.config = mutableMapOf()

"EXTRA_CAPTURE_TIMEOUT" to 120, // set Timeout to 120s, 0 to disable timeout, default = no time out
"EXTRA_CAPTURE_RESOLUTION" to 4, // Image resolution in mega pixels, default= 4MP
"EXTRA_CAPTURE_QUALITY" to 90, // JPEG compression quality, default= 90
"EXTRA_CAPTOR_SKIPPABLE" to true, // Display a SKIP buttton allowing user to skip the capture. false by default
"EXTRA_CAPTOR_CALLBACK" to captorCallback // see below Captor.Callback object

Time out

By default set at 0.0, meaning no Time out. This timer value in milliseconds can be a security feature, by setting a time frame to execute the document capture before aborting to limit people trying fake capture.

Resolution

Value from 1 to 'n', is the resolution of the capture in mega pixels (horizontal size x vertical size in pixels) by default set at 2 (MP)

JPEG quality

EXTRA_CAPTURE_QUALITY: Value from 1 to 100, by default set at 90 when JPEG is a activated

Skippable

  • true to display the SKIP button to allow user to skip the capture

  • false default value, to hide the SKIP button, user must process the capture

This setting is overrided by the Profile API

Captor Callback

val captorCallback = object : Captor.Callback{
    override fun onDataCaptured(input: Input) {
        input.pages["page1"] // Document Captured
    }

    override fun onFailure(failure: Failure) {
        // Capture failed, see Error guide chapter to identify what's the reason.
    }
}

Captor.Callback is an optional config to help you to:

  1. Collect the data captured during a call of resume() API. Those data are automatically sent to server by TDI sdk after Capture, what you get here, is only for you if you need. Please be sure to keep it safe.

  2. Listening what's happening on Failure scenario, see Error guide Chapter for reference. It could be a simple Cancel by pressing Back button or tool issue.
