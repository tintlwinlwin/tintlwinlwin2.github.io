Add TDI SDK to your Android Project


Prerequisites

  • Min Android SDK version: 6 (23) Marshmallow
  • Supported architectures: ARMv7 and ARM64


Local Repository and Components

The SDK requires local Maven repository with the following content:

Component (armeAbi-v7a + arm64-v8a) Size (real) Comment
TDI 6.5Mb -
Document Capture (1) 13.7Mb IDV
Face Capture (1) 21.7Mb IDV
MRZ Capture (1) 9.2Mb IDV
NFC Capture (1) 1.6Mb IDV
Non-ID Document Capture (1) 3.5Mb TDI component
Voice Capture (1) 0.1Mb TDI component
Fingerprint Capture (1) 30Mb TDI component
Repository 83 or 87Mb (2) Total

(1) You can remove from the repository each Capture component that you don't need.
(2) Using Android App Bundles publication, final APK size will be smaller

Contracts/signature and voice captors are embedded into TDI sdk.

Here is an interactive Excel Sheet with more details by Architecture. You can select (Yes or No) the Optional components needed.


Permissions

TDI SDK needs to have some permissions granted in order to be able to gather and operate over the desired data.

Rermissions already requested by TDI SDK when needed.

<manifest
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    package="your.package.name">

    <!-- Access to TDI server -->
    <uses-permission android:name="android.permission.INTERNET" />

    <!-- Taking picture to capture Documents and Face and FingerPrint -->
    <uses-permission android:name="android.permission.CAMERA" />

    <!-- Voice captor -->
    <uses-permission android:name="android.permission.RECORD_AUDIO" />

    <!-- Check if connect to Network -->
    <uses-permission android:name="android.permission.ACCESS_WIFI_STATE"/>
    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>

    <!-- Fingerprint captor -->
    <uses-permission android:name="android.permission.VIBRATE"

    <!-- Save report - optional -->
    <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"

        [...]
</manifest>

Check Android docs to request permissions at runtime.


Add the SDK

For the purpose of this example we will be using Android Studio + Android SDK. They can be obtained here

Assuming the client application already exists in a given project, we need to include the TDI library repository as a separate module within that project.

Project build.gradle:

buildscript {
    [...]
    ext {
        [...]
        projectMavenRepository = "${rootProject.rootDir}/../repository"
    }
}

[...]

subprojects {
    repositories {
        maven { url uri(projectMavenRepository) }
        google()
        jcenter()
    }
    [...]
}

Application build.gradle:

dependencies {
    implementation ('com.gemalto.tdi:tdi_sdk:2.0.0@aar'){
        transitive = true
    }
    implementation platform('com.google.firebase:firebase-bom:27.1.0')
    [...]
}
