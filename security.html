Security


Root Detection - MEDL

SDK is able to detect if a device seems rooted with a Thalesgroup made solution. Using TDI API on a detected rooted device will throw errors.


Hook Detection - MEDL

SDK is signing important Classes methods with a Thalesgroup made solution. By doing that it won't be possible to rewrite classes or functions that have been signed to add or remove code to access information.


Protocol HTTPS & SSL Pinning

SDK is embedding Server Certificate and trust connection only coming from our Thales servers.


Client - Server Encryption - DEP

SDK will encrypt all body request before send to server with a Thalesgroup made solution. TDI Server is providing a RSA public key use for encryption of the data and can be updated without changing SDK. Each request will open a unique session to be able to encode the request body and decode server response. Decode can only decode server response, it cant be use to decode the encoded body by the SDK.
This Encryption is optional, if you want it, you will need to request it to our Backend team, they will setup your backend to have this option and you will have a new URL to provide in TDI client API's
