Fingerprint Capture


A Thales tool by TDI

Current version: 1.0.0

This captor will help you to capture user fingers for an enrollment or an authentication

Allows to:

  • capture and extract fingerprint minutia for user enrollment & authentication
  • Check the user liveness (passive mode), no additional action by the user is required
  • Help the user during capture using notificationz, dynamic cursor and fingers position status
  • can capture:
    • all left hand fingers
    • all right hand fingers
    • all left hand fingers with an option to switch
    • all right hand fingers with an option to switch

Requirements

  • a minimum Android API level of 16

  • a rear camera with a minimum resolution of 5 MP

  • a rear LED for illumination

  • a License is required for this captor

on application build.gradle add the licence key:

android {
    ...
    defaultConfig {
        .... 
        // Fingerprint captor
        manifestPlaceholders = [VERIDIUM_SDK_LICENSE: "MIIPSQIBAzCCDwIG  .... nzjyDvSSyn1RrfAgMBhqA="]

on application manifest.xml add metadata:

    <application
        .....
        >
        <meta-data
            android:name="com.veridiumid.sdk.LICENSE"
            android:value="${VERIDIUM_SDK_LICENSE}" />

At first launch the captor needs data connection to validate the licence The licence is linked to an applicationId and has an expiration date

Customization

Captors settings

​ Captors can get a config Map to set different capture parameters.

TdiMobileSdk.getInstance().tdiMobileSdk.captorProvider["Voice"]?.config = mutableMapOf()
"EXTRA_FP_TITLE" to "Exportation", // Change the main title. by default depend of Capture mode
"EXTRA_FP_HELP" to true, // Display or not the instructional panel before a capture. true by default
"EXTRA_FP_FINGER" to 2, // Fingers to capture. FOUR_F_LEFT_SWITCHABLE by default
"EXTRA_FP_TEMPLATE" to true, // Template formats: see bellow
"EXTRA_CAPTURE_LIVENESS" to true, // Activate or not the liveness passive mode. true by default
"EXTRA_FP_LIVENESS_FACTOR" to 80, // Set the liveness factor. 80 by default
"EXTRA_CAPTURE_TIMEOUT" to 120, // set Timeout to 120s, 0 to disable timeout, default = 120s
"EXTRA_CAPTOR_CALLBACK" to captorCallback // see below Captor.Callback object

Template formats

Captor can output the following formats for capture actions:

  • VERIDFP - Veridium proprietary fingerprint format VeridiumFP. (Default)

  • NIST - ANSI/NIST ITL 1-2007, National Institute of Standards and Technology's Data Format for the Interchange of Fingerprint, Facial, & Other Biometric Information.

  • Interpol - International Criminal Police Organization INT-I : Interpol’s implementation for I-24/7 secure global police network.

  • JSON - All prints are contained within a single JSON file which contains Base64 encoded WSQ format images and can be made to contain PNG format images.

In addition, it can contain an Audit Image (a low resolution, print obscured, version of the captured image) for the purpose of human audit.

  • JSON output also supports writing of multiple scales; 500dpi; 500dpi + 15%, 500dpi - 15%. To enable the multiple scales, apply

  • ISO2005 - ISO/IEC 19794-4 (2005). Note, this is an image container

  • ZIP - DEBUG ONLY.

Fingers to capture

The following modes are available:

0 = FOUR_F_LEFT_ENFORCED - all left hand fingers

1 = FOUR_F_RIGHT_ENFORCED - all right hand fingers

2 = FOUR_F_LEFT_SWITCHABLE - all left hand fingers with a user option to switch (Default)

3 = FOUR_F_RIGHT_SWITCHABLE - all right hand fingers with a user option to switch

Captor Callback

val captorCallback = object : Captor.Callback{
    override fun onDataCaptured(input: Input) {
        input.pages["page1"] // ZIP Binary fingerprint template coded in base64
    }

    override fun onFailure(failure: Failure) {
        // Capture failed, see Error guide chapter to identify what's the reason.
    }
}

Captor.Callback is an optional config to help you to:

  1. Collect the data captured during a call of resume() API. Those data are automatically sent to server by TDI sdk after Capture, what you get here, is only for you if you need. Please be sure to keep it safe.

  2. Listening what's happening on Failure scenario, see Error guide Chapter for reference. It could be a simple Cancel by pressing Back button or tool issue.
