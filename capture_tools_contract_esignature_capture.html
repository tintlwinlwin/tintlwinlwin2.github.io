Web Contract / Signature Capture


A tool to display contract and process an eSignature

Current version: NA

No License required.

This captor allows to:

  • Visualize a Contract to be accepted by the user
  • Drawing user signature
  • Authenticate the user using:
    • email validation
    • OTP validation using SMS

The SDK is having for that 2 Captors mode SYNC & ASYNC that will be used automatically when your are using the API session.resume(taskID) and we offer some customization.

  • SYNC = Tdi sdk will open a webview to load the contract to be signed
  • ASYNC = Tdi sdk will invite you to check your email or SMS to open the contract link within your web browser.


Captors

val tdiMobileSdk = TdiMobileSdk.getInstance()
tdiMobileSdk.captorProvider["SyncWebSignature"]      // Captor will launch a webview to display the contract
tdiMobileSdk.captorProvider["AsyncWebSignature"]     // Captor will set a message to user for instruction


Customization

​ Android Strings: strings.xml

​ Android Styles: styles.xml

​ Captors configuration:

​ Captors can get a config Map to set different capture parameters.

TdiMobileSdk.getInstance().tdiMobileSdk.captorProvider["SyncWebSignature"]?.config = mutableMapOf(
    "EXTRA_CAPTURE_CONTRACT_URL" to "http://www.website.com", // The contract if not using Profile API
    "EXTRA_CAPTOR_SKIPPABLE" to true, // Display a SKIP button allowing user to skip the capture. false by default
    "EXTRA_CAPTOR_CALLBACK" to captorCallback // see below Captor.Callback object
)

TdiMobileSdk.getInstance().tdiMobileSdk.captorProvider["AsyncWebSignature"]?.config = mutableMapOf(
    "EXTRA_CAPTURE_ASYNC_TEXT_DELAY" to 2, // Timer to change instruction message
    "EXTRA_CAPTOR_SKIPPABLE" to true, // Display a SKIP button allowing user to skip the capture. false by default
    "EXTRA_CAPTOR_CALLBACK" to captorCallback // see below Captor.Callback object
)

Sync contract URL

This option is only if you don't use Profile API, else the SDK will find the URL automatically, no need to set it.

Async Timer

The Async Captor will set an first Instruction message, by default: "Please follow instruction sent to your email, keep this app in the background WARNING: DO NOT CLOSE APP" and after 10sec it will display a second Instruction message: "Please click Done when you finished the signing process"

This option will change the time in between both messages.

Default messages can be defined in your Profile API and/or strings.xml file provided above

Captor Callback

val captorCallback = object : Captor.Callback{
    override fun onDataCaptured(input: Input) {
        input.pages["page1"] // Front Document
        input.pages["page2"] // Back Document (if any)
    }

    override fun onFailure(failure: Failure) {
        // Capture failed, see Error guide chapter to identify what's the reason.
    }
}

Captor.Callback is an optional config to help you to:

  1. Collect the data captured during a call of resume() API. Those data are automatically sent to server by TDI sdk after Capture, what you get here, is only for you if you need. Please be sure to keep it safe.

  2. Listening what's happening on Failure scenario, see Error guide Chapter for reference. It could be a simple Cancel by pressing Back button or tool issue.
